# Простейшее клиент-серверное приложение

## Сценарий работы

1. Запуск сервера.

Задача сервера - раздача некоторого файла клиентам.
Имя файла для раздачи передаётся в качестве аргумента командной строки.

2. Запуск клиента.

Задача клиента - получение файла от сервера.
Имя целевого файла для клиента передаётся в качестве аргумента командной строки.

3. Обнаружение сервера клиентом.

Сервер ожидает подключения по TCP на адресе 127.0.0.1 на порту 1337.

4. Передача данных.

Сервер одновременно производит передачу данных по нескольким соединениям.
Здесь возникает задача мультиплексирования потоков данных.
